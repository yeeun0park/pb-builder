# 파리바게뜨 실제 상세페이지 구조 분석

로컬 스냅샷(`references/*.html`)을 HTML 레벨에서 해부한 결과.
Gemini 프롬프트 및 `layout_design.md`의 근거 데이터.

## 핵심 사실 — 레이아웃 DNA

| 항목 | new-paran-label | cafe-de-dessert | pb-blue-label-matcha |
|---|---|---|---|
| 본문 풀블리드 이미지 수 (desktop 기준) | **18** | 6 | 7 |
| 본문 HTML 텍스트 섹션 수 | **0** | 0 | 0 |
| 페이지 전체 HTML 텍스트 | `<h1>` 제목 1줄 + 기간 1줄 | 동일 | 동일 |

**즉, 실제 PB 페이지는 100% "풀블리드 이미지 시퀀스"이고, 텍스트(슬로건·혜택·재료 설명 등)는 전부 이미지 안에 타이포그래피로 구워져 있다.**

## 실제 제목 예시

- `NEW! 오늘도 건강하게 파란라벨 하세요💙`
- (카페·말차도 유사 패턴: "브랜드 액션 + 이모지")

## 실제 이미지 비율 분포

| 페이지 | 비율 범위 | 패턴 |
|---|---|---|
| 파란라벨 | 0.82:1 ~ **11.04:1** | 매우 좁은 수평 띠(텍스트 배너) + 일반 와이드 |
| 카페드디저트 | 0.75:1 ~ 2.33:1 | 세로 제품샷 + 와이드 |
| 블루라벨말차 | 0.45:1 ~ 7.31:1 | 초세로 + 초가로 혼재 |

**함의**: 이미지 비율이 극단적으로 다양함(세로 2:1, 가로 11:1). 따라서 우리 앱은 **이미지 원본 비율을 절대 크롭하지 않고 그대로 쌓아야** 실제 페이지 리듬을 재현 가능.

## 우리 앱의 렌더링 원칙 (이 분석 기반)

1. **기본 = fullbleed**. 이미지가 N장 있으면 N개 fullbleed 섹션 생성
2. **textBlock은 최대 1~2개만**. 인트로(hero 다음) + 선택적 클로징
3. **이미지 사이에 text 끼워넣기 금지** — 실제 PB엔 없음
4. **Hero headline = 슬로건 스타일 짧은 한 줄 + 이모지(선택)**
5. **split / grid는 특수 상황 한정** (단일 제품 1:1 매칭, 동일 카테고리 4+ 나열)
6. **이미지 높이는 원본 비율로 자동**, 크롭 금지

## 사용자 업로드 vs 실제 PB 차이

| 항목 | 실제 PB | 우리 앱 사용자 |
|---|---|---|
| 이미지 안의 텍스트 | 디자이너가 구워 넣음 | 보통 없음(제품 사진만) |
| 설명글 | 존재하지 않음 | 긴 텍스트로 제공 |
| 해결책 | - | Hero 헤드라인(슬로건화)·인트로 textBlock으로 흡수, 이미지는 그대로 풀블리드 시퀀스 |
